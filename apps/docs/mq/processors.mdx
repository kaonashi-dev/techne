---
title: "Processors"
description: "@Queue, @MqProcessor, @MqProcess, and @InjectMq turn a class into a typed queue worker."
---

A processor is an `@Injectable()` class decorated with `@Queue(name)`.
The simplest case is a single `handle()` method — no method decorator required.

```ts
import { Queue } from "@kaonashi-dev/techne/mq";
import type { MqQueue, Job } from "@kaonashi-dev/techne/mq";

@Queue("emails")
class EmailProcessor {
  // Auto-registered as the default handler for any job on this queue.
  async handle(job: Job<{ to: string; subject: string }>) {
    await sendEmail(job.data);
    return { sent: true };
  }
}
```

Register the processor through your feature's `providers` (or the `providers`
array on `TechneFactory.create()`) and declare the queue through the `mq()`
plugin — see [MQ overview](/mq/overview).

## Per-job-name dispatch

Use `@MqProcess(jobName)` to route specific job names to dedicated methods.
When a named handler exists it always takes precedence over `handle()`.

```ts
import { Queue, MqProcess } from "@kaonashi-dev/techne/mq";
import type { Job } from "@kaonashi-dev/techne/mq";

@Queue("emails")
class EmailProcessor {
  @MqProcess("welcome")
  async sendWelcome(job: Job<{ to: string }>) {
    await sendWelcomeEmail(job.data);
  }

  @MqProcess("reset")
  async sendReset(job: Job<{ to: string; token: string }>) {
    await sendPasswordReset(job.data);
  }

  // Falls through here for any job name not matched above.
  async handle(job: Job) {
    console.warn("Unhandled email job:", job.name);
  }
}
```

## Publishing

Inject the typed queue into any provider with `@InjectMq(name)`:

```ts
import { InjectMq } from "@kaonashi-dev/techne/mq";
import type { MqQueue } from "@kaonashi-dev/techne/mq";

@Injectable()
export class UsersService {
  constructor(@InjectMq("emails") private readonly emails: MqQueue<{ to: string }>) {}

  async signup(input: SignupInput) {
    const user = await this.repo.create(input);
    await this.emails.add("welcome", { to: user.email });
    return user;
  }
}
```

## `@MqProcessor` — the lower-level alias

`@Queue` is an alias for `@MqProcessor`. Both apply the same metadata and are
interchangeable. Use `@MqProcessor` when you want to be explicit that a class
is a low-level processor rather than a domain-layer queue handler.
