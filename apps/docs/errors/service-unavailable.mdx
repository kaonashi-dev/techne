This page documents the Techne RFC 7807 problem document for HTTP `service-unavailable` errors.

A `problem+json` response with `type` of
`https://github.com/kaonashi-dev/techne/blob/main/docs/errors/service-unavailable.md`
indicates the server encountered a Service Unavailable condition.

```json
{
  "type": "https://github.com/kaonashi-dev/techne/blob/main/docs/errors/service-unavailable.md",
  "title": "Service Unavailable",
  "status": 503,
  "detail": "<human-readable explanation>",
  "code": "<optional stable machine-readable code>",
  "instance": "<request URL path>",
  "requestId": "<uuid>"
}
```

During [graceful shutdown](/health/graceful-shutdown), Techne replies with
`503` to every new request while in-flight work drains. For the full error
contract and `HttpException` reference, see [Exceptions](/http/exceptions).
